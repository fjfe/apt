Es un hecho conocido que las entradas de selección de archivos son difíciles de diseñar de la manera que los desarrolladores quieren, por lo que muchos simplemente las ocultan y crean un botón que abre el cuadro de diálogo de selección de archivos. Hoy en día, sin embargo, tenemos una forma aún más elegante de manejar la selección de archivos: arrastrar y soltar.

Técnicamente, esto ya era posible porque la mayoría (si no todas) las implementaciones de la entrada de selección de archivos le permitían arrastrar archivos sobre ella para seleccionarlos, pero esto requiere que realmente muestre el elemento file. Entonces, usemos las API que nos brinda el navegador para implementar un selector y cargador de archivos mediante arrastrar y soltar.

En este artículo, usaremos ES2015+JavaScript “vanilla” (sin marcos ni bibliotecas) para completar este proyecto, y se asume que tiene un conocimiento práctico de JavaScript en el navegador. Este ejemplo, además de la sintaxis ES2015+, que se puede cambiar fácilmente a la sintaxis ES5 o transpilar por Babel, debería ser compatible con todos los navegadores Evergreen más IE 10 y 11.

Veamos lo que vas a hacer:

Eventos de arrastrar y soltar

Lo primero que debemos discutir son los eventos relacionados con arrastrar y soltar porque son la fuerza impulsora detrás de esta función. En total, hay ocho eventos que lanza el navegador relacionadas con arrastrar y soltar: drag, dragend, dragenter, dragexit, dragleave, dragover, dragstart, y drop. Nosotros no los usaremos todos porque drag, dragend, dragexit, y dragstart se disparan contra el elemento que está siendo arrastrado, y en nuestro caso, vamos a arrastrar archivos desde nuestro sistema de archivos (en lugar de elementos DOM), por lo que estos eventos nunca aparecerán.

Si tienes curiosidad acerca de ellos, puedes leer  documentación sobre estos eventos en MDN.

Como era de esperar, puedes registrar controladores de eventos para estos eventos de la misma manera que registras controladores de eventos para la mayoría de los eventos del navegador: via addEventListener.

Puedo elegir el aspecto del resaltado de código fuente ⇒    

let dropArea = document.getElementById('drop-area')

dropArea.addEventListener('dragenter', handlerFunction, false)
dropArea.addEventListener('dragleave', handlerFunction, false)
dropArea.addEventListener('dragover', handlerFunction, false)
dropArea.addEventListener('drop', handlerFunction, false)

Esta tabla describe lo que hacen estos eventos, utilizando el código de muestra dropArea para aclararlo:

Evento   ¿Cuándo se dispara?
dragenter El elemento arrastrado se arrastra sobre dropArea, lo que lo convierte en el destino del evento de colocación si el usuario lo deja caer allí.
dragleave El elemento arrastrado se arrastra fuera de dropArea y se coloca en otro elemento, convirtiéndolo en el destino del evento de caída.
dragover Cada pocos cientos de milisegundos, mientras el elemento arrastrado está sobre dropArea y se está moviendo.
drop El usuario suelta el botón del mouse y coloca el elemento arrastrado en dropArea.

Tenga en cuenta que si el elemento arrastrado se arrastra sobre un hijo de dropArea, se activará dragleave en dropArea y se activará dragenter en ese elemento hijo porque es el nuevo target. El evento drop se propagará hasta dropArea (a menos que un detector de eventos diferente detenga la propagación antes de que llegue allí), por lo que se disparará en dropArea a pesar de que no sea el target del evento.

Tenga en cuenta también que para crear interacciones personalizadas de arrastrar y soltar, deberá llamar a event.preventDefault() en cada uno de los oyentes para estos eventos. Si no lo hace, el navegador terminará abriendo el archivo que soltó en lugar de enviarlo al controlador de eventos drop.

Configurando nuestro formulario

Antes de comenzar a agregar la funcionalidad de arrastrar y soltar, necesitaremos un formulario básico con una entrada estándar file. Técnicamente, esto no es necesario, pero es una buena idea proporcionarlo como alternativa en caso de que el usuario tenga un navegador sin soporte para la API de arrastrar y soltar.

<div id="drop-area">
  <form class="my-form">
    <p>Upload multiple files with the file dialog or by dragging and dropping images onto the dashed region</p>
    <input type="file" id="fileElem" multiple accept="image/*" onchange="handleFiles(this.files)">
    <label class="button" for="fileElem">Select some files</label>
  </form>
</div>

Estructura bastante simple. Es posible que observe un controlador onchange en el input. Veremos eso más tarde. También sería una buena idea añadir una action al form y un botón submit para ayudar a aquellas personas que no tienen habilitado JavaScript. Luego, puede usar JavaScript para deshacerse de ellos y obtener una forma más limpia. En cualquier caso, usted va a necesitar un script del lado del servidor para aceptar la carga, si es algo desarrollado internamente, o que está usando un servicio como  Cloudinary que lo haga por usted.

¿Qué es Cloudinary?
 Cloudinary es un servicio en la nube que nos permitirá almacenar nuestras imágenes en su servidor y disponer de ellas mediante una url personalizada. ¿Cuál es la diferencia entre alojarlas en cloudinary o en mi propio servidor? Sus principales características:
  • Uso de CDN
    Cloudinay utiliza el Akamai's worldwide CDN. Esta red de distribución de contenidos permite que el usuario obtenga nuestros recursos del servidor más cercano al que nos los pide, optimizando el tiempo de entrega de los mismos en comparación con tenerlos alojados sólo en un servidor.
  • Cacheo de recursos
    Cloudinary implementa un excelente cacheo de los recursos que evita tener que buscarlos en su servidor con cada petición, mejorando el tiempo de carga de los mismos.
  • Optimización de calidad de las imágenes
    Añadiendo el parámetro "q_" seguido de un número a la url de nuestra imagen, podremos obtenerla en la calidad que necesitemos. En este ejemplo: http://res.cloudinary.com/di0rx0uqf/image/upload/q_80/v1464695758/pag1/foto.jpg
    obtendríamos nuestra imagen al 80% de calidad. En la mayoría de ocasiones, podremos bajar la calidad al 70%-80% sin apenas notar cambios aparentes en su aspecto, pero sí en su tamaño.
  • Transformación automática de formato
    Añadiendo el parámetro "f_auto" a nuestra url de la siguiente forma: http://res.cloudinary.com/di0rx0uqf/image/upload/f_auto/v1464695758/pag1/foto.jpg
    cloudinary nos devolverá el formato más óptimo en función del navegador de nuestro cliente. La imagen se devuelve automáticamente en formato WebP para navegadores Chrome y jpeg-xr para Internet Explorer.
Gracias a estas utilidades, podremos mejorar la fluidez de nuestra aplicación.

Añadiendo la funcionalidad de arrastrar y soltar

Ahora llegamos al meollo de la situación: arrastrar y soltar. Vamos a incluir un script en la parte inferior de la página, o en un archivo separado, como desee. Lo primero que necesitamos en el script es una referencia al área de colocación para que podamos adjuntarle algunos eventos:

let dropArea = document.getElementById('drop-area')

Ahora agreguemos algunos eventos. Comenzaremos agregando controladores a todos los eventos para evitar comportamientos predeterminados y evitar que los eventos aumenten más de lo necesario:

;['dragenter', 'dragover', 'dragleave', 'drop'].forEach(eventName => {
  dropArea.addEventListener(eventName, preventDefaults, false)
})

function preventDefaults (e) {
  e.preventDefault()
  e.stopPropagation()
}

Ahora agreguemos un indicador para que el usuario sepa que de hecho ha arrastrado el elemento sobre el área correcta usando CSS para cambiar el color del borde del área de colocación. Los estilos ya deberían estar debajo del selector #drop-area.highlight, así que usemos JS para agregar y eliminar esa clase highlight cuando sea necesario.

;['dragenter', 'dragover'].forEach(eventName => {
  dropArea.addEventListener(eventName, highlight, false)
})

;['dragleave', 'drop'].forEach(eventName => {
  dropArea.addEventListener(eventName, unhighlight, false)
})

function highlight(e) {
  dropArea.classList.add('highlight')
}

function unhighlight(e) {
  dropArea.classList.remove('highlight')
}

Tuvimos que usar ambos dragenter y dragover para resaltar debido a lo que mencioné antes. Si comienza pasando el cursor directamente sobre dropArea y luego sobre uno de sus hijos, se activará dragleave y se eliminará el resaltado. El evento dragover se dispara después de los eventos dragenter y dragleave, por lo que el resaltado se agregará nuevamente a dropArea antes de que veamos que se elimina.

También eliminamos el resaltado cuando el elemento arrastrado sale del área designada o cuando suelta el elemento.

Ahora todo lo que tenemos que hacer es averiguar qué hacer cuando se eliminan algunos archivos:

dropArea.addEventListener('drop', handleDrop, false)

function handleDrop(e) {
  let dt = e.dataTransfer
  let files = dt.files

  handleFiles(files)
}

Esto no nos acerca a la finalización, pero hace dos cosas importantes:

  1. Demuestra cómo obtener los datos de los archivos que se descartaron.
  2. Nos lleva al mismo lugar en el que file input estaba con su controlador onchange: esperando handleFiles handleFiles.

Tenga en cuenta que files no es una matriz, sino una FileList. Entonces, cuando implementemos handleFiles, necesitaremos convertirlo en una matriz para iterar sobre él más fácilmente:

function handleFiles(files) {
  ([...files]).forEach(uploadFile)
}

Eso fue anticlimático. Empecemos uploadFile por lo importante.

function uploadFile(file) {
  let url = 'YOUR URL HERE'
  let formData = new FormData()

  formData.append('file', file)

  fetch(url, {
    method: 'POST',
    body: formData
  })
  .then(() => { /* Done. Inform the user */ })
  .catch(() => { /* Error. Inform the user */ })
}

Aquí utilizamos FormData una API de navegador incorporada para crear datos de formularios para enviarlos al servidor. Luego usamos la API fetch para enviar la imagen al servidor. Asegúrese de cambiar la URL para que funcione con su back-end o servicio, y formData.append cualquier dato de formulario adicional que pueda necesitar para brindarle al servidor toda la información que necesita. Alternativamente, si desea admitir Internet Explorer, es posible que desee usar XMLHttpRequest, lo que significa que uploadFile se vería así:

function uploadFile(file) {
  var url = 'YOUR URL HERE'
  var xhr = new XMLHttpRequest()
  var formData = new FormData()
  xhr.open('POST', url, true)

  xhr.addEventListener('readystatechange', function(e) {
    if (xhr.readyState == 4 && xhr.status == 200) {
      // Done. Inform the user
    }
    else if (xhr.readyState == 4 && xhr.status != 200) {
      // Error. Inform the user
    }
  })

  formData.append('file', file)
  xhr.send(formData)
}

Dependiendo de cómo esté configurado su servidor, es posible que desee verificar diferentes rangos de números status en lugar de solo 200, pero para nuestros propósitos, esto funcionará.

Funciones adicionales

Esa es toda la funcionalidad básica, pero a menudo queremos más funcionalidad. Específicamente, en este tutorial, agregaremos un panel de vista previa que muestra al usuario todas las imágenes elegidas, luego agregaremos una barra de progreso que permite al usuario ver el progreso de las cargas. Entonces, comencemos con la vista previa de imágenes.

Vista previa de la imagen

Hay un par de formas de hacer esto: puede esperar hasta que se haya cargado la imagen y pedirle al servidor que envíe la URL de la imagen, pero eso significa que debe esperar y las imágenes pueden ser bastante grandes a veces. La alternativa, que exploraremos hoy, es usar la  API FileReader en los datos de archivo que recibimos del evento. Esto es asincrónico y, alternativamente, podría usar  FileReaderSync, pero podríamos estar intentando leer varios archivos grandes seguidos, por lo que esto podría bloquear el hilo durante bastante tiempo y arruinar realmente la experiencia. Así que creemos una función previewFile y veamos cómo funciona:

function previewFile(file) {
  let reader = new FileReader()
  reader.readAsDataURL(file)
  reader.onloadend = function() {
    let img = document.createElement('img')
    img.src = reader.result
    document.getElementById('gallery').appendChild(img)
  }
}

Aquí creamos un new FileReader lo llamamos readAsDataURL con el objeto File. Como se mencionó, esto es asincrónico, por lo que debemos agregar un controlador de eventos onloadend para obtener el resultado de la lectura. Luego usamos la URL de datos base 64 como src para un nuevo elemento de imagen y lo agregamos al elemento gallery. Solo hay dos cosas que deben hacerse para que esto funcione ahora: agregue el elemento gallery y asegúrese de que realmente se llame a previewFile.

Primero, agregue el siguiente HTML justo después del final de la etiqueta form:

<div id="gallery"></div>

Nada especial; es solo un div. Los estilos ya están especificados para él y las imágenes que contiene, por lo que no queda nada por hacer allí. Ahora cambiemos la función handleFiles a la siguiente:

function handleFiles(files) {
  files = [...files]
  files.forEach(uploadFile)
  files.forEach(previewFile)
}

Hay algunas formas en que podría haber hecho esto, tales como composición o una callback a forEach que ejecuta uploadFile y previewFile en ella, pero esto también funciona. Y con eso, cuando suelta o selecciona algunas imágenes, deberían aparecer casi instantáneamente debajo del formulario. Lo interesante de esto es que, en ciertas aplicaciones, es posible que en realidad no desee cargar imágenes, sino almacenar las URL de datos de ellas en localStorage o en algún otro caché del lado del cliente para que la aplicación pueda acceder más tarde. Personalmente, no puedo pensar en ningún buen caso de uso para esto, pero estoy dispuesto a apostar que hay algunos.

Seguimiento del progreso

Si algo puede tardar un tiempo, una barra de progreso puede ayudar al usuario a darse cuenta de que realmente se está progresando y dar una indicación de cuánto tiempo tardará en completarse. Agregar un indicador de progreso es bastante fácil gracias a la etiqueta HTML5 progress. Comencemos agregando eso al código HTML esta vez.

<progress id="progress-bar" max=100 value=0></progress>

Puede colocarlo justo después de label o entre form el div galería, lo que más le apetezca. De hecho, puede colocarlo donde desee dentro de las etiquetas body. No se agregaron estilos para este ejemplo, por lo que mostrará la implementación predeterminada del navegador, que es útil. Ahora trabajemos en agregar JavaScript. Primero veremos la implementación usando fetch y luego mostraremos una versión para XMLHttpRequest. Para empezar, necesitaremos un par de variables nuevas en la parte superior del script:

let filesDone = 0
let filesToDo = 0
let progressBar = document.getElementById('progress-bar')

Cuando usamos fetch, solo podemos determinar cuándo finaliza una carga, por lo que la única información que rastreamos es cuántos archivos se seleccionaron para cargar (en filesToDo) y la cantidad de archivos que se terminaron de cargar (en filesDone). También mantenemos una referencia al elemento #progress-bar para que podamos actualizarlo rápidamente. Ahora creemos un par de funciones para administrar el progreso:

function initializeProgress(numfiles) {
  progressBar.value = 0
  filesDone = 0
  filesToDo = numfiles
}

function progressDone() {
  filesDone++
  progressBar.value = filesDone / filesToDo * 100
}

Cuando comencemos a cargar, se llamará a initializeProgress para restablecer la barra de progreso. Luego, con cada carga completa, llamaremos a progressDone para incrementar el número de cargas completadas y actualizaremos la barra de progreso para mostrar el progreso actual. Así que llamemos a estas funciones actualizando un par de funciones antiguas:

function handleFiles(files) {
  files = [...files]
  initializeProgress(files.length) // <- Add this line
  files.forEach(uploadFile)
  files.forEach(previewFile)
}

function uploadFile(file) {
  let url = 'YOUR URL HERE'
  let formData = new FormData()

  formData.append('file', file)

  fetch(url, {
    method: 'POST',
    body: formData
  })
  .then(progressDone) // <- Add `progressDone` call here
  .catch(() => { /* Error. Inform the user */ })
}

Y eso es. Ahora echemos un vistazo a la implementación XMLHttpRequest. Podríamos simplemente hacer una actualización rápida a uploadFile, pero en realidad XMLHttpRequest nos brinda más funcionalidad que fetch, es decir, podemos agregar un detector de eventos para el progreso de la carga en cada solicitud, que periódicamente nos dará información sobre la cantidad de la solicitud finalizada. Debido a esto, necesitamos realizar un seguimiento del porcentaje de finalización de cada solicitud en lugar de solo cuántas se realizaron. Entonces, comencemos reemplazando las declaraciones de filesDone y filesToDo con lo siguiente:

let uploadProgress = []

Entonces también necesitamos actualizar nuestras funciones. Vamos a renombrar progressDone como updateProgress y a cambiar lo siguiente:

function initializeProgress(numFiles) {
  progressBar.value = 0
  uploadProgress = []

  for(let i = numFiles; i > 0; i--) {
    uploadProgress.push(0)
  }
}

function updateProgress(fileNumber, percent) {
  uploadProgress[fileNumber] = percent
  let total = uploadProgress.reduce((tot, curr) => tot + curr, 0) / uploadProgress.length
  progressBar.value = total
}

Ahora initializeProgress inicializa una matriz con una longitud igual a numFiles, que está llena de ceros, lo que indica que cada archivo está completo al 0%. En updateProgress averiguamos qué imagen está actualizando su progreso y cambiamos el valor en ese índice al percent proporcionado. Luego calculamos el porcentaje de progreso total tomando un promedio de todos los porcentajes y actualizamos la barra de progreso para reflejar el total calculado. Seguimos llamando a initializeProgress en handleFiles de la misma manera que hicimos en el ejemplo fetch, por lo que ahora todo lo que necesitamos actualizar es uploadFile para llamar a updateProgress.

function uploadFile(file, i) { // <- Add `i` parameter
  var url = 'YOUR URL HERE'
  var xhr = new XMLHttpRequest()
  var formData = new FormData()
  xhr.open('POST', url, true)

  // Add following event listener
  xhr.upload.addEventListener("progress", function(e) {
    updateProgress(i, (e.loaded * 100.0 / e.total) || 100)
  })

  xhr.addEventListener('readystatechange', function(e) {
    if (xhr.readyState == 4 && xhr.status == 200) {
      // Done. Inform the user
    }
    else if (xhr.readyState == 4 && xhr.status != 200) {
      // Error. Inform the user
    }
  })

  formData.append('file', file)
  xhr.send(formData)
}

Lo primero a tener en cuenta es que agregamos un parámetro i. Este es el índice del archivo en la lista de archivos. No necesitamos actualizar handleFiles para pasar este parámetro porque está usando forEach, lo que ya da el índice del elemento como segundo parámetro para las devoluciones de llamada. También agregamos el detector de eventos progress a xhr.upload para que podamos llamar a updateProgress con el progreso. El objeto de evento (al que se hace referencia en el código con e) tiene dos piezas de información pertinentes: loaded que contiene la cantidad de bytes que se han cargado hasta ahora y total que contiene la cantidad de bytes que tiene el archivo en total.

La operación || 100 está ahí porque a veces, si hay un error, e.loaded y e.total serán cero, lo que significa que el cálculo saldrá como NaN, por lo que 100 se utiliza en su lugar, informando que el archivo está hecho. También puede utilizar 0. En cualquier caso, el error aparecerá en el controlador readystatechange para que pueda informar al usuario sobre ellos. Esto es simplemente para evitar que se produzcan excepciones por intentar hacer operaciones matemáticas con NaN.

Conclusión

Esa es la pieza final. Ahora tiene una página web donde puede cargar imágenes mediante arrastrar y soltar, obtener una vista previa de las imágenes que se cargan de inmediato y ver el progreso de la carga en una barra de progreso. Puede ver la versión final (con XMLHttpRequest) en acción en  CodePen, pero tenga en cuenta que el servicio al que subo los archivos tiene límites, por lo que si muchas personas lo prueban, es posible que se rompa por un tiempo.

¿Qué es SortableJS?
 SortableJS es una biblioteca de JavaScript que permite crear fácilmente listas reordenables arrastrando y soltando elementos. La biblioteca ofrece una gran flexibilidad y facilita a los usuarios colocar los elementos en el orden deseado.

Su caso básico de uso (lista de elementos que se pueden reordenar arrastrándolos y soltándolos) se implementa con 3 pasos:
  1. Crear una lista en HTML (Por ej. con ID "lista"). 
    <div id="lista">
  <div> Un elemento </div>
  <div> Otro </div>
  <div> Más... </div>
</div>
  2. Referenciarla desde JS. 
    var lista = document.getElementById('lista');
  3. Convertirla en lista ordenable usando Sortable. 
    new Sortable(lista, {
    animation: 150
    // Otras opciones...
});
En el caso de necesitar dos listas con elementos arrastrables ente ellas, les pondremos el mismo grupo a ambas. 
new Sortable(lista1, {
    group: 'grupoA',    // ambas listas tienen el mismo grupo
    animation: 150
});

new Sortable(lista2, {
    group: 'grupoA',
    animation: 150
});
Para implementar casos más complejos, es recomendable probar su  demo y revisar su 📓 documentación.
