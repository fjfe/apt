Cadena de bloques
Una cadena de bloques, conocida en inglés como blockchain, es una estructura de datos cuya información se agrupa en conjuntos (bloques) a los que se les añade metainformaciones [hash] relativas a otro bloque de la cadena anterior en una línea temporal. De esta forma, gracias a técnicas criptográficas [criptografía], la información contenida en un bloque solo puede ser repudiada o editada modificando todos los bloques posteriores. Esta propiedad permite su aplicación en un entorno distribuido [BD distribuida] de manera que la estructura de datos blockchain puede ejercer de base de datos pública no relacional que contenga un histórico irrefutable de información.

  • ¿Cómo se respeta el orden de los bloques?
    Anotando en la cabecera de cada uno el hash del anterior.
  • ¿Cómo se protege la integridad de la cadena?
    Con el hash. Alterar uno obliga a alterar todos los siguientes.
  • ¿Cómo se evitan colisines al escribir el siguiente bloque?
    Con el azar: Obligando a 'minar' el fragmento que consigue 30 ceros al inicio del hash.

banquero (en el sentido "conserva las cuentas de los usuarios").

Web 1.0, Web 2.0 y Web 3.0 explicados

Narottam04   Publicado el 24/dic/2021

Todos estamos de acuerdo en que la web ha cambiado drásticamente a lo largo de los años, tanto como consumidor como como desarrollador. Gracias a los avances en la web, producir sitios web y aplicaciones atractivos nunca ha sido tan fácil.

El propósito de esta publicación de blog es explorar el pasado de la web, así como discutir el futuro potencial de la web, es decir, la web 3.0.

Web 1.0 Beta

El concepto de hipertexto es anterior a la World Wide Web en varias décadas. Sin embargo, casi todos los sistemas de hipertexto se basaron en archivos locales. Tim Berners-Lee deseaba desarrollar un sistema que pudiera funcionar a través de redes, permitiendo a las personas vincularse de un archivo en una máquina a otro archivo en otra.

Entonces, en diciembre de 1990, codificó una aplicación con  Objective-C  para su próximo sistema informático y la nombró World Wide Web.

WorldWideWeb era más que una simple aplicación de exploración de archivos. Era tanto un navegador como un editor.

Puede consultar el primer navegador web de todo el mundo:  https://worldwideweb.cern.ch/

Web 1.0

Web 1.0 fue la primera etapa de la revolución de la World Wide Web, generalmente conocida como   web de solo lectura  . Los sitios web eran informativos y solo contenían contenido estático con hipervínculos o, en palabras simples, no había CSS, enlaces dinámicos, interactividad como iniciar sesión en los usuarios, comentarios en las publicaciones del blog, etc.

Los sitios web se crearon utilizando  Server Side Include   o   Common Gateway Interface   (CGI) en lugar de una aplicación web escrita en un lenguaje de programación dinámico como Perl, PHP, Python o Ruby.

En la era de la Web 1.0, es decir, de 1991 a 2004, los usuarios de Internet eran  consumidores  de contenido creado por  creadores de contenido  .

Web 2.0

Web 2.0 desde 2004 hasta ahora, es la segunda etapa de la revolución de la World Wide Web, generalmente conocida como   web de lectura y escritura.  Se hizo hincapié en el contenido generado por los usuarios, la facilidad de uso, la cultura participativa y la interoperabilidad.

Algunas de las primeras plataformas basadas en web 2.0 son YouTube, Facebook, Amazon, etc. Debido a las tecnologías de CMS como WordPress, los blogs, la creación de una tienda de comercio electrónico se volvió extremadamente popular.

La Web 2.0 provocó un cambio fundamental, donde las personas pudieron compartir sus perspectivas, opiniones, pensamientos y experiencias a través de una variedad de herramientas y plataformas en línea.

Desventajas de la Web 2.0

Antes de analizar las desventajas, veamos cómo funciona una aplicación web 2.0 tradicional.

Un cliente (usuario) realizará una solicitud HTTP al servidor y, si todo está correcto, enviará esa página web como respuesta.

Un defecto importante aquí es que todos los datos se almacenan en un servidor centralizado, controlado por las empresas.

Facebook, Google y Twitter comenzaron a almacenar datos de los usuarios en sus servidores para que puedan brindarnos un mejor contenido a través del aprendizaje automático. Esto, a su vez, haría que permanezcamos en sus sitios web por más tiempo, lo que proporcionaría más ingresos publicitarios para estas empresas.

Las empresas finalmente comenzaron a vender nuestra información a los anunciantes, ¡lo que significó más dinero para ellos!

Las ofertas de intercambio de datos de Facebook expuestas

Para resumir la web 2.0 podemos decir que los  usuarios son el producto.

Web 3.0

El concepto web 3.0 tiene como objetivo crear una Internet descentralizada, pero segura, en la que las personas puedan intercambiar dinero e información de forma segura, sin necesidad de intermediarios o grandes empresas tecnológicas.

A diferencia de la Web 2.0, donde los datos se almacenan en una sola base de datos o en un proveedor en la nube, las aplicaciones Web 3 se ejecutan en blockchain, nodos de igual a igual (servidores).

A diferencia de la web 2.0, donde usted es el producto, algunas personas predicen que en la web 3.0, usted será el propietario del contenido. Según los especialistas de web 3.0, la corporación estará dirigida por un grupo descentralizado conocido como  DAO (Organización Autónoma Descentralizada)  eliminando la necesidad de CEO y alta gerencia en una empresa. Debido a que su identidad digital no está ligada a su identidad real en la web 3.0, puede permanecer anónimo en Internet sin dejar de vivir su vida habitual.

Un ejemplo de aplicación web 3.0 es Obsidian para Whatsapp, Odyessy, LBRY para YouTube, etc.

¿Vale la pena el revuelo que rodea a la Web 3.0?

Varias personas tienen diferentes perspectivas sobre esto; algunos creen que es un engaño creado por comerciantes de criptomonedas, mientras que otros creen que es un verdadero negocio. En mi opinión, blockchain y web 3.0 aún se encuentran en una etapa temprana de desarrollo y tienen un largo camino por recorrer.

Algunos argumentos actuales válidos a partir de los escritos son:

  • El problema de la computación y el ancho de banda para escalar blockchain.
    Web3 is Bullshit
  • La minería no es ecológica y consume mucha energía para minar y verificar transacciones.
    Artículo de la Universidad de Cambridge al respecto
    Bitcoin causing annual CO2 emissions comparable to those of Vegas: Study
  • Opinión personal:  Crear una plataforma de redes sociales no regulada, como YouTube o Facebook, donde los usuarios pueden agregar cualquier dato pero no se puede borrar, podría ser un arma de doble filo. Por un lado, tiene libertad de expresión, pero por otro lado, puede haber personas que publiquen contenido explícito que nadie puede eliminar. Esto puede ser muy peligroso a menos que exista un medio para verificar si la información contiene contenido explícito por máquina. Me encantaría conocer tu opinión sobre este tema.
