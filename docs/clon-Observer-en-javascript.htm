Patrón Observer en JavaScript

Original: Tempus Code · Por Chema Alfonso · 24/oct/2019→

En este artículo hablaremos de patrones de diseño, y más concretamente del  patrón observer. Aprenderemos a realizar su implementación mediante un ejemplo sencillo y fácil de entender, a partir del cual podrás implementar este diseño en tus proyectos para mejorar la calidad de tu código.

El patrón Observer, es un patrón de diseño de software que nos permite definir una dependencia de uno a muchos entre elementos del sistema, los cuales serán notificados por parte del observable cada vez que se realice una modificación.

Para ver todo esto de forma clara lo realizaremos mediante un ejemplo en el que tendremos 1 input maestro (Original) el cual disparará, tras cada tecla pulsada, el método de nuestro observable que notificará a cada uno de sus subscriptores. En este caso serán otros 3 inputs que escucharán los cambios (3 Plagios) mediante el observable.    

// Definición de la clase Observable
class Observable{

    // El constructor de la clase inicializa un arreglo vació
    // donde se almacenaran los observadores.
    constructor(){
        this.observers = [];
    }

    // Método para añadir nuevos observadores.
    subscribe( observer ){
        this.observers.push(observer);
    }

    // Método para eliminar observadores.
    unsubscribe( observer ){
        this.observers = this.observers.filter( subscriber => subscriber != observer);
    }

    // Método para notificar los cambios a los observadores.
    notify( data ){
        this.observers.forEach( observer => observer( data ) );
    }
}

// Referenciamos los elementos del DOM
// con los que queremos trabajar.
const elmMaster = document.getElementById('elmMaster');
const elm  = document.getElementById('elm1');
const elm2 = document.getElementById('elm2');
const elm3 = document.getElementById('elm3');

// Asignamos la función a realizar cuando
// se dispare el observable a una variable
// (En este caso asignaríamos la data al valor del input).
const updateFirstInput  = (data) => elm.value  = data;
const updateSecondInput = (data) => elm2.value = data;
const updateThirdInput  = (data) => elm3.value = data;

// Instanciamos nuestra clase observable
const inputsObserver = new Observable();

// Suscribimos las funciones que se ejecutarán
// al dispararse el observable
inputsObserver.subscribe(updateFirstInput);
inputsObserver.subscribe(updateSecondInput);
inputsObserver.subscribe(updateThirdInput);

// Añadimos el evento keyup a nuestro input maestro
// para ejecutar el resultado tras soltar cada tecla
// y llamamos a la función notify de nuestro observable
// mandando como parámetro los datos a actualizar.
elmMaster.addEventListener('keyup', (event) => inputsObserver.notify(event.target.value));

Esto es todo lo necesario para suscribirse y poder utilizar los datos introducidos en elmMaster en tiempo real.

Para dejar de ser notificados con algún elemento, basta con llamar al método unsubscribe con parámetro el elemento que queremos eliminar de la suscripción.

// Esto terminará con la escucha del observable 
// por parte de updateSecondInput.
inputsObserver.unsubscribe(updateSecondInput);

Espero que con este ejemplo hayas aprendido algo más sobre patrones de diseño y te quede más clara la forma de crear un observable y su utilización e implementación en lenguaje JavaScript.

Te animo a aprender algunas buenas prácticas en JavaScript.

    function e(s){return document.getElementById(s);}       // Elemento con id=s

    class Observable{           // Definición de la clase Observable

      constructor(){            // Crea array vacío que almacenará los observadores
        this.observers = [];
      }

      subscribe( observer ){    // Añadir nuevos observadores
        this.observers.push(observer);
      }

      unsubscribe( observer ){  // Eliminar observadores
        this.observers = this.observers.filter(subscriber => subscriber != observer);
      }

      notify( data ){           // Notificar los cambios a los observadores
        this.observers.forEach( observer => observer( data ) );
      }
    }


    window.addEventListener('load', function(){   // inicia con DOM cargado
      e('tit').innerHTML = document.title;

      // Referenciamos los elementos del DOM con los que queremos trabajar
      const tOrig = e('tOrig');
      const tPla1 = e('tPla1');
      const tPla2 = e('tPla2');
      const tPla3 = e('tPla3');

      // Funciones que se ejecutarán cuando se dispare el observable
      // (En estos casos asignaríamos la data al valor del input 'Plagio N')
      const actPla1 = (data) => tPla1.value = data;
      const actPla2 = (data) => tPla2.value = data;
      const actPla3 = (data) => tPla3.value = data;

      // Instanciamos nuestra clase observable
      const obs = new Observable();

      // Suscribimos las funciones que se ejecutarán al dispararse el observable
      obs.subscribe(actPla1);
      obs.subscribe(actPla2);
      obs.subscribe(actPla3);

      // El evento keyup en input original (se ejecutará tras soltar cada tecla)
      // llama al método notify del observable con parámetro los datos a actualizar
      tOrig.addEventListener('keyup', (event) => obs.notify(event.target.value));

      // Uso casillas de verificación para suscribir/desuscribir cada plagio
      cSus1.addEventListener('change', function(){
        if (cSus1.checked)
          obs.subscribe(actPla1);
        else
          obs.unsubscribe(actPla1);
      });
      cSus2.addEventListener('change', function(){
        if (cSus2.checked)
          obs.subscribe(actPla2);
        else
          obs.unsubscribe(actPla2);
      });
      cSus3.addEventListener('change', function(){
        if (cSus3.checked)
          obs.subscribe(actPla3);
        else
          obs.unsubscribe(actPla3);
      });
    });
